= 取得功能狀態
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `GetFeatureStatus`獲取聚類特徵狀態的方法。



== 範圍

此方法具有以下輸入參數：

[cols="1a,3a,1a,1a,1a"]
|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
特徵
 a| 
集群特徵的狀態。如果沒有提供值，系統將傳回所有功能的狀態。可能的值：

* `Vvols`：檢索 VVols 叢集功能的狀態。
* `SnapMirror`：檢索SnapMirror複製叢集功能的狀態。
* `Fips`：檢索 HTTPS 通訊功能的 FIPS 140-2 加密狀態。
* `FipsDrives`：檢索 FIPS 140-2 磁碟機加密功能的狀態。
* `VolumeLoadBalanceOnActualIOPS`：根據實際 IOPS 而非最小 IOPS 功能檢索切片均衡狀態。從 Element 12.8 版本開始可用。

 a| 
細繩
 a| 
沒有任何
 a| 
不

|===


== 傳回值

此方法傳回以下值：

|===
| Name | 描述 | 類型 


 a| 
特徵
 a| 
一個要素物件數組，表示要素名稱及其狀態。對象成員：

* feature: (字串) 功能的名稱。
* 已啟用：（布林值）該功能是否已啟用。

 a| 
JSON 物件數組

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
  "method": "GetFeatureStatus",
    "params": {
    },
    "id": 1
}
----


== 回應範例

此方法傳回類似以下範例的回應：

[listing]
----
{
    "id": 1,
    "result": {
        "features": [
            {
                "enabled": true,
                "feature": "Vvols"
            },
            {
                "enabled": true,
                "feature": "SnapMirror"
            },
            {
                "enabled": true,
                "feature": "Fips"
            },
            {
                "enabled": true,
                "feature": "FipsDrives"
            },
            {
                "enabled": true,
                "feature": "VolumeLoadBalanceOnActualIOPS"
            }
        ]
    }
}
----
