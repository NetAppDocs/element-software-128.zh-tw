= 建立備份目標
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以使用 `CreateBackupTarget`建立並儲存備份目標訊息，這樣每次建立備份時就不需要重新輸入該資訊。



== 參數

此方法有以下輸入參數：

|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
姓名
 a| 
備份目標名稱。
 a| 
細繩
 a| 
沒有任何
 a| 
是的



 a| 
屬性
 a| 
JSON 物件格式的名稱-值對清單。
 a| 
JSON 物件
 a| 
沒有任何
 a| 
是的（但可以是空的）

|===


== 傳回值

此方法傳回以下值：

|===


| Name | 描述 | 類型 


 a| 
備份目標ID
 a| 
為新備份目標分配唯一識別碼。
 a| 
整數

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
   "method": "CreateBackupTarget",
   "params": {
      "name": "mytargetbackup"
   },
"id": 1
}
----


== 回應範例

此方法傳回類似以下範例的回應：

[listing]
----
{
   "id": 1,
   "result": {
      "backupTargetID": 1
   }
}
----
