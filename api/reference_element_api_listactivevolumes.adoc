= 列出活動卷
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `ListActiveVolumes`取得系統中目前活動卷清單的方法。磁碟區列表按 VolumeID 順序排序，可以分成多部分（頁）返回。



== 參數

此方法有以下輸入參數：

|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
包含虛擬卷
 a| 
預設情況下，回應中包含虛擬磁碟區。若要排除虛擬卷，請設定為 false。
 a| 
布林值
 a| 
真的
 a| 
不



 a| 
起始卷 ID
 a| 
開始返回 VolumeID。如果不存在具有此 VolumeID 的捲，則使用 VolumeID 順序的下一個磁碟區作為清單的開頭。若要翻頁瀏覽列表，請傳遞上一個回應中最後一個磁碟區的 VolumeID + 1。
 a| 
整數
 a| 
0
 a| 
不



 a| 
限制
 a| 
要傳回的最大磁碟區資訊物件數。  0（零）返回所有磁碟區（無限制）。
 a| 
整數
 a| 
（無限）
 a| 
不

|===


== 傳回值

此方法傳回以下值：

|===


| Name | 描述 | 類型 


 a| 
卷
 a| 
活動卷列表。
 a| 
xref:reference_element_api_volume.adoc[體積]大批

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
   "method": "ListActiveVolumes",
   "params": {
      "startVolumeID"  : 0,
      "limit" : 1000
   },
   "id" : 1
}
----
