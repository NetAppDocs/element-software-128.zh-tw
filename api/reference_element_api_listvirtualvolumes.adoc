= 列出虛擬卷
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `ListVirtualVolumes`列出系統中目前虛擬磁碟區的方法。您可以使用此方法列出所有虛擬卷，或僅列出其中的一部分。



== 參數

此方法有以下輸入參數：

|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
細節
 a| 
回覆的詳細程度。可能的值：

* 正確：請在回覆中包含有關每個 VVol 的更多詳細資訊。
* 錯誤：在回覆中包含有關每個 VVol 的標準詳細資訊等級。

 a| 
布林值
 a| 
錯誤的
 a| 
不



 a| 
限制
 a| 
要列出的虛擬卷的最大數量。
 a| 
整數
 a| 
10000
 a| 
不



 a| 
遞迴
 a| 
指定是否在回應中包含有關每個 VVol 子節點的資訊。可能的值：

* 正確：請在回覆中包含每個 VVol 的子女的資訊。
* 錯誤：請勿在回應中包含有關每個 VVol 的子項的資訊。

 a| 
布林值
 a| 
錯誤的
 a| 
不



 a| 
啟動虛擬磁碟區 ID
 a| 
回應中清單的起始虛擬磁碟區 ID。
 a| 
UUID類型
 a| 
沒有任何
 a| 
不



 a| 
虛擬磁碟區 ID
 a| 
若要檢索資訊的虛擬磁碟區 ID 清單。如果省略此參數，則該方法將僅傳回有關這些虛擬磁碟區的資訊。
 a| 
虛擬磁碟區 ID UUID 數組
 a| 
沒有任何
 a| 
不

|===


== 傳回值

此方法具有以下傳回值：

|===


| Name | 描述 | 類型 


 a| 
下一個虛擬磁碟區 ID
 a| 
清單中下一個虛擬磁碟區的 ID。
 a| 
唯一識別符



 a| 
虛擬磁碟區
 a| 
描述系統中目前虛擬磁碟區的物件清單。
 a| 
xref:reference_element_api_virtualvolume.adoc[虛擬磁碟區]大批

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
  "method": "ListVirtualVolumes",
    "params": {
    },
    "id": 1
}
----
