= 列出集群故障
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `ListClusterFaults`用於列出集群上檢測到的任何故障資訊的方法。透過這種方法，您可以列出目前存在的故障以及已解決的故障。系統每 30 秒快取一次故障。



== 參數

此方法有以下輸入參數：

|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
最佳實踐
 a| 
包括由次優系統配置引發的故障。可能的值：

* 真的
* 錯誤的

 a| 
布林值
 a| 
沒有任何
 a| 
不



 a| 
故障類型
 a| 
確定傳回的故障類型。可能的值：

* 目前：列出所有未解決的活動故障。
* 已解決：列出先前已偵測並已解決的故障。
* 全部：列出目前故障和已解決的故障。您可以在故障物件的「`resolved`」成員中查看故障狀態。

 a| 
細繩
 a| 
全部
 a| 
不

|===


== 傳回值

此方法傳回以下值：

|===


| Name | 描述 | 類型 


 a| 
故障
 a| 
描述所請求的叢集故障的物件。
 a| 
xref:reference_element_api_fault.adoc[過失]

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
  "method": "ListClusterFaults",
    "params": {
      "faultTypes": "current",
      "bestPractices": true
    },
    "id": 1
}
----
