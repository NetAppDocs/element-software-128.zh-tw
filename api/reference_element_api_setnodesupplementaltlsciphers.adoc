= 設定節點補充TLS密碼
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
你可以使用 `SetNodeSupplementalTlsCiphers`指定補充 TLS 密碼清單的方法。您可以在管理節點上使用此命令。



== 範圍


NOTE: 必須對管理節點呼叫此方法。例如：

[listing]
----
https://<management node IP>:442/json-rpc/10.0
----
此方法具有以下輸入參數：

|===
| Name | 描述 | 類型 | 預設值 | 必需的 


 a| 
補充密碼
 a| 
補充密碼套件名稱採用 OpenSSL 命名方案。密碼套件名稱不區分大小寫。
 a| 
細繩
 a| 
沒有任何
 a| 
是的

|===


== 傳回值

此方法具有以下傳回值：

|===
| Name | 描述 | 類型 


 a| 
強制密碼
 a| 
節點必須使用的TLS密碼套件清單。這些密碼始終在節點上處於活動狀態。
 a| 
細繩



 a| 
補充密碼
 a| 
節點的補充 TLS 密碼套件清單。
 a| 
細繩

|===


== 請求範例

該方法的請求類似於以下範例：

[listing]
----
{
   "method": "SetNodeSupplementalTlsCiphers",
   "params": {
        "supplementalCiphers": [
            "DHE-RSA-AES128-SHA256",
            "DHE-RSA-AES128-GCM-SHA256",
            "ECDHE-RSA-AES128-SHA256",
            "ECDHE-RSA-AES128-GCM-SHA256"
        ]
   },
   "id": 1
}
----


== 回應範例

此方法傳回類似以下範例的回應：

[listing]
----
{
  "id" : 1,
  "result" : {
        "mandatoryCiphers": [
            "DHE-RSA-AES256-SHA256",
            "DHE-RSA-AES256-GCM-SHA384",
            "ECDHE-RSA-AES256-SHA384",
            "ECDHE-RSA-AES256-GCM-SHA384"
        ],
        "supplementalCiphers": [
            "DHE-RSA-AES128-SHA256",
            "DHE-RSA-AES128-GCM-SHA256",
            "ECDHE-RSA-AES128-SHA256",
            "ECDHE-RSA-AES128-GCM-SHA256"
        ]
    }
}
----
