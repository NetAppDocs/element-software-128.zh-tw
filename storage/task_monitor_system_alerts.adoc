= 查看系統警報
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以查看有關叢集故障或系統錯誤的警報資訊。警報可以是資訊、警告或錯誤，是衡量叢集運作狀況的良好指標。大多數錯誤會自動解決。

您可以使用 ListClusterFaults API 方法來自動監控警報。這樣您就可以收到所有警報的通知。

. 在 Element UI 中，選擇「報告」>「警報」。
+
系統每 30 秒刷新頁面上的警報。

+
對於每個事件，您都會看到以下資訊：

+
|===


| 物品 | 描述 


 a| 
ID
 a| 
與叢集警報關聯的唯一ID。



 a| 
嚴重程度
 a| 
警報的重要性程度。可能的值：

** 警告：一個小問題，可能很快就需要處理。系統升級仍然允許。
** 錯誤：可能導致效能下降或失去高可用性 (HA) 的故障。一般而言，這些錯誤不應影響服務的其他方面。
** 嚴重故障：影響服務的嚴重故障。系統無法處理 API 或客戶端 I/O 請求。在這種狀態下運行可能會導致資料遺失。
** 最佳實務：未使用建議的系統配置最佳實務。




 a| 
類型
 a| 
受故障影響的部件。可以是節點、磁碟機、叢集、服務或磁碟區。



 a| 
節點
 a| 
此故障所指節點的節點 ID。包含節點和驅動器故障，否則設定為 -（短橫線）。



 a| 
磁碟機 ID
 a| 
此故障所指的磁碟機的磁碟機 ID。包含驅動器故障，否則設定為 -（短橫線）。



 a| 
錯誤代碼
 a| 
用於描述故障原因的描述性代碼。



 a| 
細節
 a| 
故障的描述及更多細節。



 a| 
日期
 a| 
故障記錄的日期和時間。

|===
. 點選「顯示詳情」查看單一警報的相關資訊。
. 要查看頁面上所有警報的詳細信息，請按一下「詳細資料」列。
+
系統解決警報後，有關該警報的所有資訊（包括解決日期）都會移至「已解決」區域。
